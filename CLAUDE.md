# zkm-pdf

zkm plugin (handle: `pdf`) that imports text-extractable PDFs into the knowledge
store. Scans the store inbox for `.pdf` symlinks deposited by upstream plugins
(zkm-eml) and optionally walks an external `source_dir`. Extracts text with
`pypdf`, emits `pdfs/YYYY/MM/<date>_<slug>.md`, dedups by SHA-256, and skips
scanned-only PDFs (no text layer) so `zkm-scan` can OCR them later.

Independent git repo living at `~/src/zkm/plugins/zkm-pdf/` inside the zkm core
checkout (polyrepo convention — see core `CLAUDE.md`). Versioned independently,
tags `vX.Y.Z`, loose-0.x SemVer, bump-and-tag in the same commit.

See `ARCHITECTURE.md` for design decisions with rationale and rejected
alternatives. See `ROADMAP.md` for the executor-facing task queue.

## Commands

```bash
uv sync --extra dev          # env (Python 3.11+; dev extra adds pytest, ruff, fpdf2)
uv run pytest                # full suite (hermetic — tmp_path stores, no network)
uv run pytest -k <expr>      # one test / one roadmap item's done-check
uv run ruff check .          # lint (line-length 100, py311 target)
uv run python tests/build_fixtures.py   # regenerate committed tests/fixtures/*.pdf
```

There is no standalone CLI. The plugin is invoked through core:
`zkm convert pdf` (requires this repo discovered under `<zkm>/plugins/zkm-pdf/`
or installed as a wheel via `uv tool install zkm --with zkm-pdf`).

## Layout

```
src/zkm_pdf/
├── convert.py      # THE implementation (all logic lives here)
├── __init__.py     # re-exports convert (wheel entry-point target: zkm.plugins → zkm_pdf)
└── plugin.yaml     # packaged copy — MUST stay in sync with root plugin.yaml
convert.py          # root shim for core's filesystem discovery (dev workflow);
                    # core injects src/ onto sys.path (SB2), shim re-exports zkm_pdf.convert
plugin.yaml         # root copy read by filesystem discovery — keep in sync with src copy
tests/
├── conftest.py     # sys.path insert for the root shim; tests otherwise import zkm_pdf
├── build_fixtures.py   # fpdf2/pypdf generator for committed fixtures (placeholder content only)
├── fixtures/*.pdf      # committed synthetic PDFs — NEVER add real personal documents
└── test_*.py
```

## Configuration

Bare snake_case keys in `<store>/zkm-config.yaml` under the plugin section
(M2 config migration, 2026-05 — the old `.env` / `PDF_*` SCREAMING keys are gone):

| Key | Default | Description |
|---|---|---|
| `source_dir` | *(empty)* | Optional external directory walked recursively for `.pdf` files (read-only) |
| `min_text_chars` | `100` | Min extracted chars to emit md; below → skip + JSONL audit log. **Provisional heuristic** (see ROADMAP id:9475) |

## Gotchas (hard-won; do not rediscover)

- **Dual `plugin.yaml`**: the file exists at repo root (filesystem discovery)
  AND at `src/zkm_pdf/plugin.yaml` (wheel). Any change must be applied to BOTH,
  byte-identical. Version bumps touch `pyproject.toml`, both `plugin.yaml`
  copies, and `PLUGIN_VERSION` in `src/zkm_pdf/convert.py` — four places.
- **Root `convert.py` is a shim**, not the implementation. It exists only so
  core's filesystem discovery can load the plugin without a wheel install. If
  you add public names tests or core need, re-export them through the shim too.
- **Config keys are bare snake_case** (`source_dir`, `min_text_chars`). Tests
  or docs using `PDF_SOURCE_DIR` / `PDF_MIN_TEXT_CHARS` are stale relics of the
  pre-M2 era — fix them on sight.
- **Editable zkm dep**: `[tool.uv.sources] zkm = { path = "../.." }` — `uv sync`
  only works when this repo sits at `<zkm-core>/plugins/zkm-pdf/` (true for the
  dev checkout and for relay worktrees hosted inside a core worktree).
- **Fixtures are placeholder-only**: synthetic PDFs generated by
  `tests/build_fixtures.py`. Regenerate via the script, never hand-craft, never
  commit real documents (published-repos-must-be-generic rule).
- **Skipped PDFs are logged, not lost**: `<store>/.zkm-state/zkm-pdf-skipped.jsonl`
  records every below-threshold skip for later review / zkm-scan handoff.
- **`zkm convert pdf` runs under core guards**: dirty-tree guard
  (`ZKM_BYPASS_DIRTY_CHECK=1` in tests/dev) and concurrent-run guard (exit 75,
  `ZKM_BYPASS_RUN_GUARD=1`). The plugin's own pytest suite bypasses core
  entirely by calling `convert()` directly.

## Conventions

- Python 3.11+, `uv`-managed, ruff (E/F/I/UP-ish core style, line-length 100), pytest.
- Conventional commits; English code/docs; ISO 8601 dates.
- Minimize dependencies: `pypdf` + `python-frontmatter` + core `zkm` helpers only.
  Object-storage primitives (`zkm.atomic`, `zkm.cas`, `zkm.sidecar`, `zkm.inbox`,
  `zkm.hashing`) are imported from core — never reimplemented here.
